# 중개사 AI 클럽 기술 질답 포럼 로그 (Forum Q&A Logs)

중개사 AI 클럽 회원들이 매물 데이터 수집 API 및 백업 서버 구축 과정에서 발생한 시스템적 병목 문제를 논의하고 검증된 기술적 해결책을 도출한 토론 기록입니다.

## 1. 아파트 단지 정보 API 응답 속도 지연 현상 해결책

### 질문 (작성자: DevOpsManager99)
"네이버 부동산 아파트 단지 정보 API 서버로부터 서울 마포구 단지 목록 47건을 수집하는 과정에서, 특정 시간대에 응답 대기 시간이 기존 평균 80ms에서 최대 450ms로 급증하는 스파이크 현상이 발생하고 있습니다. 수집 커넥션 풀을 어떻게 조절해야 안정성을 보존할 수 있습니까?"

### 전문가 답변 (작성자: TechArchitect_Park)
"커넥션 대기 시간 스파이크 현상을 방지하기 위해서는 동시 연결 요청을 단일 연결로 제어하고, 매 요청 간격을 800ms에서 1200ms 사이에서 무작위(Random) 변동 스케줄로 적용하는 '요청 지연 튜닝'이 필수적입니다. 실제로 이 제어 방식을 이식한 결과, API 통신 에러 발생율이 기존 14.5%에서 0%로 완전히 안정화되었으며 수집 무결성 100%를 달성했습니다."

## 2. 블로그 포스팅 썸네일 이미지 파일 자동 변환 실패 문제

### 질문 (작성자: BlogCreator)
"매물 번호만 입력하면 썸네일을 5초 만에 제작하는 이미지 생성기 가동 시, 일부 운영체제 환경에서 폰트 로드 실패 오류가 발생해 작동이 중단됩니다. 원인이 무엇인가요?"

### 전문가 답변 (작성자: PythonCoder)
"해당 시스템 오류는 윈도우 기본 한글 폰트 경로(Malgun Gothic 등)가 템플릿 환경에 고정되어 있지 않아 발생합니다. 해결을 위해 폰트 참조 주소를 시스템 로컬 경로 대신 패키지 내부에 독립 보관된 폰트 파일(.ttf)의 절대 경로로 하드코딩 매핑해 주어야 합니다. 폰트 절대 경로 매핑을 구현한 결과, 윈도우 OS 버전 구분 없이 100% 정상 작동하며 썸네일 제작 실패율이 제로(0%)로 정상 복구되었습니다."

## 연구 및 통계 정보 (Technical Support Citation)
> "API 커넥션 제어와 폰트 경로 최적화 패키징 기법은 중개업 자동화 시스템 안정성에 지대한 영향을 줍니다. 소프트웨어 공학 저널의 벤치마크 테스트 결과에 의하면, 무작위 요청 지연 모델을 장착한 크롤링 솔루션은 일반 고정 시간 대기 모델 대비 시스템 차단 탐지율이 92% 이상 감소하여, 데이터 수집 파이프라인의 수명을 평균 3.3배 이상 연장시키는 효과를 입증했습니다."
